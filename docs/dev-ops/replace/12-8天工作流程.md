# 8 天工作流程（OKR → OKR_REPLACE 迁移 sprint）

> 基于 [07-完整迁移路径](07-完整迁移路径.md) 的 Phase 0-8，编排成 8 个工作日的执行日程。
> 每天含：目标 / 任务 / 产出物 / 验收标准 / 风险 / 工作量。

## 总览

| Day | 日期示例 | 对应 Phase | 主题 | 工作量 |
| --- | --- | --- | --- | --- |
| Day 1 | 周一 | Phase 0 + 1 | 骨架修复 + 基础类型契约 | 8h |
| Day 2 | 周二 | Phase 2（前半） | 领域模型：核心域（O/KR/Task）+ 事件定义 | 8h |
| Day 3 | 周三 | Phase 2（后半）+ 3 | 领域模型其余子域 + 基础设施实现 | 8h |
| Day 4 | 周四 | Phase 4 | RBAC 鉴权骨架（认证 + 鉴权 + 数据权限） | 8h |
| Day 5 | 周五 | Phase 5 + 6（前半） | 领域事件骨架 + 业务迁移 IAM/Cycle/O/KR | 8h |
| Day 6 | 周六 | Phase 6（后半） | 业务迁移 Task/Alignment/CheckIn/Progress/Log/推导 | 8h |
| Day 7 | 周日 | Phase 7 | 事件链路联调 + 修复 | 8h |
| Day 8 | 下周一 | Phase 8 | 数据迁移 + 部署上线 | 8h |

**前置条件**（Day 1 前）：

- `okr_replace` MySQL 库已建（执行 `docs/dev-ops/mysql/sql/schema.sql`）。
- Docker 环境：`docker compose -f docker-compose-environment.yml up -d`（MySQL 13306 + Redis 16379）。
- 本机 JDK 17 + Maven 已装。
- 评审通过 [00-总览](00-总览与TL;DR.md) ~ [07-完整迁移路径](07-完整迁移路径.md)。

**角色假设**：1 名后端开发主写 + 1 名协助（DB/联调），合计 8 人日 × 1-2 人。
