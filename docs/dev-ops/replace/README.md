## 关键事实速览

1. **架构形态**：9 微服务 + RabbitMQ → 单体 DDD + Spring 进程内领域事件。
2. **写法借鉴**：抄 ai-mcp-gateway 的七模块分层 + Repository 端口反转；但 ai-mcp-gateway **没有领域事件实现**（`trigger/listener/` 是空目录，仅 `package-info` 注释提示方向），事件部分基于 OKR 原链路重新落地。
3. **新增 RBAC**：启用 OKR"建了表没启用"的五表 + `data_scope`，做认证/鉴权/数据权限三层。
4. **资产复用**：OKR_REPLACE 已有 17 个 DAO/PO/mapper XML + 16 表 schema，直接用 Repository 适配，不重写持久层。
5. **阻断项**：Phase 0 必先修——根 pom case 重复声明 + case 自循环依赖 + dev 库名 + Redis 缺失。
