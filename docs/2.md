# 后端待办清单（OKR 功能迭代）

> 目标：把当前已开始的 OKR 后端能力按优先级拆成可执行任务，方便逐步实现与验收。

## 当前状态
- `KR / Task / Alignment / Progress / Check-in / Log` 的基础表、Mapper、DTO、Repository、Service 已开始铺设。
- 代码仍需要做一次完整收口：接口签名统一、Case 编排接管、核心业务闭环联调、单测/集成测补齐。
- 现阶段最重要的是先打通 **KR -> Objective 进度汇总**，再做 **Task / Alignment / Check-in / Log**。

## P0：必须先完成

### 1. KR CRUD + 进度计算
- [ ] 补全 `okr_key_result` 的 CRUD 接口与分页查询。
- [ ] 定义 `KR completion_rate` 更新后触发 Objective 进度重算。
- [ ] 明确权重汇总规则：`Objective.progress = Σ(KR.rate * weight) / Σ(weight)`。
- [ ] 在 `OkrObjectiveService` 或独立进度服务中实现统一重算入口。
- [ ] 增加 KR 相关单测：新增 / 更新 / 删除后进度是否正确。
- [ ] 在 Trigger 层补齐 `OkrKeyResultController` 的接口说明与参数校验。

### 2. OKR 目标修改权限收口
- [ ] 将“可见范围”和“可修改范围”彻底拆开。
- [ ] 列表查询保留 `queryVisibleUserIds`。
- [ ] 更新 / 删除改为使用 `queryEditableUserIds`。
- [ ] 校验下级可见上级，但不可修改上级 OKR。
- [ ] 统一在 `OkrObjectiveService.checkOwnership` 中做最终判定。

## P1：核心业务闭环

### 3. Task CRUD + 任务-用户关联
- [ ] 完成 `okr_task` 的 CRUD 与按 KR 查询接口。
- [ ] 完成 `okr_task_user` 关联增删改查。
- [ ] 任务创建时校验对应 KR 是否在可编辑范围内。
- [ ] 任务状态更新时写入进度记录与日志。
- [ ] 增加“我的任务 / 部门任务 / 所有任务”查询接口。

### 4. 目标对齐（Alignment）
- [ ] 完成 `okr_objective_alignment` 的创建、删除、查询。
- [ ] 增加对齐树查询接口，支持前端展示上下游关系。
- [ ] 增加防环校验，避免目标形成循环对齐。
- [ ] 约定清楚对齐类型：目标对齐目标 / 目标对齐 KR。
- [ ] 在 Objective 删除前处理引用关系。

## P2：历史与审计

### 5. Progress Record（进度变更历史）
- [ ] 统一定义 `target_type`：OBJECTIVE / KR / TASK。
- [ ] 所有进度变化都走 `okr_progress_record`。
- [ ] KR 更新、Task 状态变更、Objective 汇总更新时都写流水。
- [ ] 提供按目标 / 时间 / 操作人分页查询接口。

### 6. Check-in（周期复盘）
- [ ] 完成 `okr_check_in` 主表 CRUD。
- [ ] 完成 `okr_check_in_item` 明细 CRUD。
- [ ] 复盘提交时自动关联当前周期与目标。
- [ ] 复盘提交后同步写进度记录。
- [ ] 支持按目标、周期、用户查询复盘列表。

### 7. Operation Log（操作日志）
- [ ] 定义统一日志模型：模块、资源类型、资源 ID、动作、前后 JSON、操作者、备注。
- [ ] 在 KR / Task / Alignment / Check-in 等关键操作处落日志。
- [ ] 保证日志与业务事务一致，避免业务成功但日志丢失。
- [ ] 提供日志查询接口供审计页面使用。

## P3：流程编排与架构收口

### 8. Case 编排扩展
- [ ] 将 KR / Task / Check-in 等流程拆分为 case。
- [ ] 按节点组织：`LoadContext -> Validate -> Persist -> PostHook`。
- [ ] Trigger 层统一改为调用 case service。
- [ ] 将进度记录、操作日志等 post hook 接入 case 节点。
- [ ] 为 case 增加组合测试，保证节点顺序正确。

### 9. 接口与领域统一
- [ ] 检查 Controller / Service / Repository / DAO / Mapper 的方法名是否一致。
- [ ] 检查 DTO / VO / PO 字段映射是否一致。
- [ ] 补齐 Swagger / 文档注释。
- [ ] 统一返回码与异常码。

## 建议执行顺序
1. KR CRUD + 进度重算。
2. Task CRUD + 任务-用户关联。
3. Alignment。
4. Progress Record + Check-in。
5. Operation Log。
6. Case 编排统一接管。

## 验收标准
- Objective 的进度值能随着 KR 自动更新。
- Task 创建 / 指派 / 查询完整可用。
- 目标对齐树可视化数据返回正确。
- 进度记录、复盘、日志都能查到历史。
- 关键流程均由 case 编排驱动。
